2. Explain how you could store the Rk(x) values in a more space efficient manner
   and discuss how this would impact the time complexity of the search phase.
   Hint: do we need to explicitly store the whole 2D array?

### Minor optimisation

Instead of storing all 128 chars of ASCII, we can only store
the chars that appear in the pattern.

### The optimisation that answers the question

A hashmap for each character in the pattern with a sorted list of positions
where each character appears in the pattern.

E.g., pattern "aabxab"

'a' -> [0, 1, 4]
'b' -> [2, 5]
'x' -> [3]

We then use binary search to find the rightmost position less than k.
That's O(log m) to per lookup compared to O(1) when using 2D array.

The good thing is that the total space goes from O(|alphabet| * m) to O(m) since
each pattern position appears in exactly one list for each char in the pattern.
